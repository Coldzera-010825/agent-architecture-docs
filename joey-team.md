# 团队协同样本：Joey 主控与双专员架构

本文档基于真实工作流产生，演示了在 OpenClaw 体系下，**大管家（主控 Agent）与专精职能 Agent** 的实际协作与任务分发模式。

## 角色设定

在我们的日常流转中，确立了以下“包工头与工人”的闭环体制：
* **Cold**: 人类包工头，只负责下发大白话、非结构化的核心业务需求。
* **Joey (我)**: 团队核心主控与管家（主控 Agent）。全权接收 Cold 的指令，负责意图理解、需求拆解，并生成极具专业度和规范性的高阶 Prompt，最终分发给执行层。
* **rat**: 文本专员（专职 Sub-agent）。唯一职责是处理文本重构、排版润色、信息结构化转换与翻译。接收包含严格格式约束（如周报模板）的指令，输出纯净的成稿文本。
* **prof**: 代码专员（专职 Sub-agent）。承担代码编程、Bug 追踪、架构设计相关的实操重活。

## 实战工作流：一份周报的诞生

当人类用户抛出一堆口语化且散乱的本周和下周工作内容时，Joey 的处理流程如下：

### 1. 意图抓取与流程对位
Joey 调取本地 `MEMORY.md` 和业务线特定规则（如“51项目制作全流程”），判定该任务属于**标准化管理汇报**。

### 2. 编写高阶 Prompt (路由给 rat)
针对文本收敛的需求，Joey 不会自己干，而是将其转化为对 `rat` 下发的系统级约束。示例：
```markdown
【角色设定】：你是团队的资深文字专员 rat，长于文本重构、排版润色与信息结构化。
【输出格式约束】：严格遵循以下骨架，只输出最终成稿，不需要任何开场白或解释。
本周已完成TOP5...下周计划TOP5...
【素材输入】：（整合并提纯后的人类用户需求）
【要求】：文字必须干练、专业，有商务汇报感。请立即开始处理。
```

### 3. 子层执行与验收
* `rat` 接收任务，严格依据格式输出商务周报。
* Joey 获取结果并进行最终审验，确认无污染后，包裹后返回给 Cold。

## 协作优势

1. **认知隔离**：人类无需思考提示词工程和机器能懂的指令。
2. **术业专攻**：调度端（Joey）保留大局观与调度逻辑上下文，不需要把上下文空间浪费在具体的代码拼装或死板的造词填空上。执行端（rat/prof）收到的背景极为纯粹，不会发生角色错乱或过度发散。
3. **流程控制透明**：每一次分发前，主控 Agent 将生成的 Prompt 下发痕迹回显给人类过目，实现“黑盒变白盒”的工作透明化管理。
